[개요]
DataBase(DB)
-공유되어 사용될 목적으로 통합 저장, 관리되는 데이터
-관련된 테이블들의 집합

Table
-사물이나 개념의 본질적인 속성을 모아서 표로 만든 것
-사물이나 개념을 표현하는 단위

DBMS
-데이터베이스를 체계적으로 관리할 수 있도록 기능을 제공하는 프로그램

SQL
-DBMS를 다루기 위한 표준화된 언어 체계

[용어]
RDBMS
-2차원 테이블 형태로 데이터를 저장하고 테이블과 테이블 간의 관계를 통해 데이터를 저장 및 사용
-스키마(구조)를 먼저 정하고 스키마(구조)에 맞게 데이터를 사용 및 관리

스키마(구조)
-테이블을 만들기 위한 구조와 제약을 명세하는 것

행 == 튜플 == 레코드
-테이블의 한 행을 의미함
-행은 해당 테이블에서 구조화된 데이터 집합으로서 하나의 개채(사물, 개념)를 의미
ex) 첫 번째 학생의 모든 정보(25살 홍길동 서울), 두 번째 학생의 모든 정보(22살 이순신 서울)

열 == 어트리뷰트 == 필드
-테이블의 한 열을 의미함
-열은 해당 테이블에서 모든 개체에 대한 특정 정보(속성)의 나열을 의미
ex) 나이(25살, 22살...), 이름(홍길동, 이순신...), 거주지(대전, 서울...)


DDL(데이터 정의 언어)
-데이터베이스 구조에 관한 명령
생성 : create
조회 : show
수정 : alter
삭제 : drop

DML(데이터 조작 언어)
-생성 : insert into 테이블
-조회 : select * from 테이블
-수정 : update 테이블 set
-삭제 : delete from 테이블
